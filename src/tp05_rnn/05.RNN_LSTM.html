00

Where we are

Sessions 1–4 built powerful classifiers. They all share one blind spot.

The blind spot of everything we have built

Every model from S01 to S04 is a bag of words. TF-IDF, Naive Bayes, Logistic Regression, GloVe mean pool — they all treat a document as an unordered set of tokens. Word order is completely discarded.
Word order is lost — examples
"The dog bit the man." → same vector as "The man bit the dog."

"Not feeling happy." → dominated by happy → predicted as positive.

"I used to love this restaurant." → love fires positive despite used to.

"The food was good but the service was terrible." → mixed signal, wrong prediction.
What we need
A model that reads text left to right, one word at a time, remembering what came before.

The representation of word t should depend on words 1 through t−1. Context matters.
This is exactly what a Recurrent Neural Network does. It processes sequences token by token, maintaining a running summary — the hidden state — of everything seen so far.

Negation — a concrete failure

Negation is one of the clearest cases where word order matters. "Not happy" and "happy" have opposite meanings but almost identical bag-of-words representations.
GloVe mean pool — what it sees
Sentence: "not feeling happy"

Vectors: v(not), v(feeling), v(happy)
Mean: dominated by v(happy) — strong positive signal

Prediction: positive — wrong.
TF-IDF + LR — what it sees
Features: not=1, feeling=1, happy=1
The coefficient learned for happy is strongly positive.
The model may have seen "not happy" rarely in training.

Prediction: often wrong on negation.
What an RNN/LSTM sees
Step 1: reads not → hidden state encodes negation context
Step 2: reads feeling → updates hidden state
Step 3: reads happy → in the context of not

The hidden state at step 3 carries the information that not was seen earlier. With enough training data, the model learns that not + positive word = negative sentiment.

Prediction: negative — correct.
01

The RNN Idea

One step at a time. One vector summarises the past.

Recurrent neural network — intuition

Key idea: reuse the same weight matrix at every step. The network's weights do not change across time steps — the same transformation is applied again and again, once per token.
Folded view (one step)
xt ──┐
        ├─→ RNN cell → ht
ht-1 ─┘
The cell takes the current token x_t and the previous hidden state h_{t-1}, and produces the new hidden state h_t.
The update equation
h_t = tanh( W_h · h_{t-1} + W_x · x_t + b )
W_h: hidden-to-hidden weights
W_x: input-to-hidden weights
b: bias
tanh: squashes output to (−1, +1)
Unrolled view — sentence: "the cat sat"

output

At each step the same weights are used. h_3 summarises the entire sentence and is fed to a Dense + softmax for classification.

RNN — unrolled computation

The same cell, applied six times. One hidden state carries the past forward.

From text to sequences — the pipeline

Before feeding text to an RNN we need to convert it to a fixed-length integer sequence. Keras provides everything we need.
Step 1 — Tokenize
Assign an integer index to each word by frequency rank.

"the" → 1 (most frequent)
"cat" → 42
"sat" → 78

Tokenizer(num_words=1000) keeps the top 1000 words. Rare words → unknown token.
Step 2 — Pad
All sequences must have the same length for batching.

"the cat sat" → [1, 42, 78]
padded to length 10 → [0, 0, 0, 0, 0, 0, 0, 1, 42, 78]

Pre-padding (zeros at the start) is preferred for RNNs: the last hidden state sees real tokens, not padding.
Step 3 — Embed
Map each integer index to a dense vector.

Embedding(vocab_size, dim)

This is a learnable lookup table. Index 42 → a 16-dimensional vector, updated during training.

Alternatively: load pre-trained GloVe weights and freeze them. 
from tensorflow.keras.preprocessing.text import Tokenizer
from tensorflow.keras.preprocessing.sequence import pad_sequences

train_seqs = tokenizer.texts_to_sequences(train_texts)

02

Vanishing Gradients

Why plain RNNs forget. Why LSTM was invented.

Backpropagation through time

Training an RNN means computing gradients all the way back through every time step. This is called Backpropagation Through Time (BPTT).
The chain rule unrolled
The loss at step T depends on h_T, which depends on h_{T-1}, which depends on h_{T-2}, and so on.

The gradient of the loss w.r.t. h_0 is a product of T Jacobians:
∂L/∂h_0 = (∂h_T/∂h_{T-1}) · (∂h_{T-1}/∂h_{T-2}) · … · (∂h_1/∂h_0)
Each factor is the derivative of tanh, multiplied by the weight matrix W_h.
What goes wrong
The derivative of tanh is at most 1, usually much less (near 0 in the saturated regions).

Multiply 50 numbers each less than 1 together: 0.8 × 0.8 × … × 0.8 (×50) ≈ 0.00001

The gradient at step 0 is essentially zero. The network cannot learn from information early in the sequence. Long-range dependencies are lost.
Concrete example
Sentence: "The cat, which had been sleeping all afternoon in the warm sun by the window, sat on the mat."

The subject cat is 15 tokens before the verb sat. By the time the gradient flows back to step 1, it has been multiplied by 15 small numbers.

SimpleRNN cannot reliably link cat (singular) to the agreement of sat. LSTM can.
Note: for short sequences (SMS messages, tweets, short sentences) vanishing gradients are less of a problem. SimpleRNN works reasonably well for spam detection. It fails on long documents — news articles, novels, code.

Vanishing gradient — visualised

Gradient flows right-to-left. Each step multiplies by a number < 1. The signal dies before reaching t=1.

03

Long Short-Term Memory

Hochreiter & Schmidhuber, 1997. The gated solution to vanishing gradients.

The LSTM idea — a second memory channel

Core insight: add a second memory channel — the cell state c_t — that flows through the sequence with only small, controlled modifications. Gradients can flow back through it without repeated multiplication.
SimpleRNN — one memory channel
h_t = tanh( W_h · h_{t-1} + W_x · x_t + b )
One vector h_t does everything: stores long-term context AND serves as input to the next step. Gradient must flow through tanh repeatedly. It vanishes.
LSTM — two memory channels
c_t = f_t ⊙ c_{t-1} + i_t ⊙ c̃_t ← cell state
h_t = o_t ⊙ tanh(c_t) ← hidden state
The cell state c_t is updated by addition (not multiplication). Gradients can flow back through it almost unchanged across many steps.
An analogy — working memory vs. long-term notes
Think of h_t as your working memory right now — what you are actively thinking about.

Think of c_t as a notepad you carry through the sequence. You can write new things, erase old things, or leave it unchanged. The notepad is what lets you remember something said 20 sentences ago.
Three gates control what gets written, erased, and read from the notepad. Each gate is a vector of values between 0 and 1, learned by the network from data.

The three gates

Forget gate f_t
f_t = σ( W_f · [h_{t-1}, x_t] + b_f )
Decides what to erase from the cell state.

Values near 0 → erase that dimension.
Values near 1 → keep that dimension.

Example: when the subject of a sentence changes, the forget gate clears the old subject from memory.
Input gate i_t
i_t = σ( W_i · [h_{t-1}, x_t] + b_i )
c̃_t = tanh( W_c · [h_{t-1}, x_t] + b_c )
Decides what to write into the cell state.

i_t controls how much of the candidate c̃_t to add.

Example: when a key fact appears (a name, a negation), the input gate writes it into the cell state.
Output gate o_t
o_t = σ( W_o · [h_{t-1}, x_t] + b_o )
h_t = o_t ⊙ tanh( c_t )
Decides what to expose from the cell state as the hidden state.

The cell may store many things; only the relevant ones are exposed to the next step and the output layer.
σ is the sigmoid function (output in [0, 1]) — it acts as a soft switch. ⊙ is element-wise multiplication. All gate vectors have the same dimension as the cell state. The LSTM has 4× more parameters than SimpleRNN of the same size.

Inside the LSTM cell

Two streams: cell state c_t (long memory) and hidden state h_t (working memory). Three gates control the flow.

LSTM — worked example on spam

Trace what happens as the LSTM reads a spam message token by token. This is conceptual — the actual gate values are learned from data.
Input: "You have won a £1000 cash prize"
Token Likely gate activity
youWeak signal — nothing written
haveWeak signal
wonInput gate opens — strong spam signal written to cell
aCell state preserved by forget gate ≈ 1
£1000Input gate opens again — reinforces spam signal
cash prizeFurther reinforcement
Final h_T: strong spam representation → Dense(sigmoid) → 0.97 → spam.
Why this is better than SimpleRNN
The cell state accumulates evidence across the sequence. won, £1000, cash prize each push the cell further toward the spam direction, and the forget gate keeps those signals alive until the end.

SimpleRNN would have to pass this information through the tanh bottleneck at every step — by the time it reaches the Dense layer the signal has degraded.
In Keras — one line difference
Replace:
SimpleRNN(units=32)

With:
LSTM(units=32)

The rest of the architecture — Embedding, Dense, compile — stays identical. Keras handles all the gate computations internally.

LSTM in Keras — full classifier

from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import (
    Embedding, LSTM, Dense, Dropout
)
from tensorflow.keras.optimizers import Adam

model = Sequential([
    Embedding(
        input_dim=1000,
        output_dim=16,
        input_length=50,
        name="embedding",
    ),
    LSTM(units=32, name="lstm"),
    Dropout(0.3, name="dropout"),
    Dense(1, activation="sigmoid",
          name="output"),
])

model.compile(
    loss="binary_crossentropy",
    optimizer=Adam(learning_rate=1e-3),
    metrics=["accuracy"],
)
model.summary()
Parameter count — LSTM vs SimpleRNN
With embed_dim=16, units=32:

SimpleRNN: (16+32+1)×32 = 1,568
LSTM: 4 × (16+32+1)×32 = 6,272

4× more params — one set per gate (forget, input, output, cell candidate).
Dropout after LSTM
The LSTM has many parameters and can overfit on small datasets. Dropout(0.3) randomly zeros 30% of the hidden state during training, forcing the network to not rely on any single unit. Only active during training, not inference.

Building the GloVe embedding matrix
def build_embedding_matrix(
    word2idx: dict,
    glove: dict,
    dim: int = 50,
) -> np.ndarray:
    """
    Parameters
    ----------
    word2idx : dict
        word → integer index (1-based).
    glove : dict
        word → np.ndarray.
    dim : int
        GloVe dimension.

    Returns
    -------
    np.ndarray, shape (vocab+1, dim)
    """
    vocab_size = len(word2idx) + 1
    matrix = np.zeros((vocab_size, dim))
    for word, idx in word2idx.items():
        if word in glove:
            matrix[idx] = glove[word]
    return matrix

Transfer learning — frozen vs fine-tuned

Why freeze the GloVe weights on small datasets? The GloVe vocabulary has 400,000 words × 50 dimensions = 20 million parameters. With only 145 training sentences, trying to update all of them would immediately overfit. Freezing keeps the pre-trained semantic structure intact.
Large dataset
Millions of labelled examples.

Fine-tune the embeddings.
The domain-specific vocabulary may be different from GloVe's training corpus. Updating helps adapt.

Example: training a sentiment model on 5M Amazon reviews.
Medium dataset
Tens of thousands of examples.

Fine-tune with low learning rate, or freeze for first N epochs then unfreeze.

Example: TP-04 AG News (108K articles with GloVe).
Small dataset
Hundreds of examples.

Freeze the embeddings. Only train the LSTM + Dense layers (a few thousand parameters).

Example: TP-05B Emojifier (145 sentences).
This is the core of transfer learning: use representations learned on a large corpus, adapt only the task-specific layers on your (smaller) target dataset. You will apply this principle at a much larger scale in Session 06 with BERT.

Frozen GloVe vs trainable embedding

Two architecture choices. The right one depends on how much labelled data you have.

05

Limits and What's Next

Why RNNs are no longer the state of the art.

The parallelism problem

Fatal limitation of RNNs: the computation at step t depends on the hidden state from step t−1. Steps cannot be computed in parallel. Training on long sequences is slow.
Sequential computation
To compute h_50 you must first compute h_1, h_2, …, h_49 in sequence.

A document with 512 tokens requires 512 sequential matrix multiplications. Even with batching, the GPU is underutilised — it cannot parallelise across the time dimension.

Scaling to longer contexts (1,000+ tokens) becomes very slow.
Long-range memory is still limited
LSTM is much better than SimpleRNN, but still imperfect. Information from 200 steps ago is diluted by 200 forget gate multiplications (each slightly less than 1).

For tasks requiring reasoning over entire documents, LSTMs fall short.
The Transformer solution (S06)
Instead of processing left to right, compute attention between every pair of tokens simultaneously.

h_i attends to all other positions in one step — no sequential bottleneck. Training can be fully parallelised across the sequence.

Long-range dependencies: token 1 and token 512 are equidistant. No vanishing gradient problem across the sequence direction.
Historical arc:
S03: BoW — no order, no parameters
S04: Embeddings — semantic geometry, still no order
S05: RNN/LSTM — order preserved, sequential
S06: Transformer — order preserved, parallel, global attention
S07: BERT — pre-trained Transformer, fine-tuned
